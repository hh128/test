# E2E-Автотесты для Events Widget

Репозиторий содержит E2E-автотесты для виджета [Events Calendar], написанные на Playwright + TypeScript.

## Стек
* **Playwright** (тест-раннер и браузерный движок)
* **TypeScript** 
* **Allure** (отчеты)
* **ESLint / Prettier** (кодстайл)

## Что покрыто тестами?
1. **Смена языка (Локализация)**: проверяю, что меняется и UI, и URL (`/en/`).
2. **Отлов JS-ошибок**: повесил слушатель на `pageerror`. Если страница плюется ошибками в консоль при генерации виджета-тест упадет.
3. **API (Network/Iframe)**: при выборе тематик (например, SEO), скрипт парсит `src` сгенерированного iframe и проверяет, что туда реально прокинулись правильные параметры `event_type=...`.
4. **Data-Driven подход для тем**: вместо копипасты написал один тест в цикле, который прогоняет проверку всех 4-х цветовых схем (Green, Blue, Turqoise, Purple) из массива.
5. **Системный буфер обмена**: честная проверка кнопки "Скопировать" через API ОС `navigator.clipboard.readText()`.

Также используется паттерн **Page Object Model**, инициализация которого вынесена в кастомную **фикстуру** Playwright (`eventsFixture.ts`), чтобы в спеках не было мусора с `beforeEach`.

## Запуск 

Требуется Node.js v18+

1. Установка:
   ```bash
   npm install
   npx playwright install --with-deps
   ```

2. Запуск тестов:
   ```bash
   npm run test
   ```

3. Генерация Allure-отчета:
   ```bash
   npm run report
   ```

*(В репе также настроен простенький GitHub Action для прогона в CI)*
